# Resource (bcadmincenter_environment_update_schedule)

Manages an explicitly scheduled upgrade for a Business Central environment.

Use this resource instead of `application_version` on `bcadmincenter_environment` when you need explicit scheduling control over which version to target and when the upgrade should run.

~> **Note:** Do **not** use `application_version` on `bcadmincenter_environment` and `bcadmincenter_environment_update_schedule` for the same environment simultaneously.

~> **Warning:** Destroying this resource removes it from Terraform state only. No API call is made and the scheduled upgrade on the Business Central side is **not** cancelled.

## Important Notes

~> **Warning:** Do **not** use `application_version` on `bcadmincenter_environment` and `bcadmincenter_environment_update_schedule` for the same environment simultaneously. Choose one approach per environment.

~> **Warning:** **Destroying** this resource removes it from Terraform state only. No API call is made to cancel the scheduled upgrade on the Business Central side. The upgrade will continue as scheduled regardless of whether this resource exists in Terraform state.

-> **Note:** The Business Central Admin Center API does not support deselecting an update without selecting a new one (`"selected": false` returns an error). For this reason, the `delete` operation is a state-only removal.

## Example Usage

### Explicitly Scheduled Future Upgrade

```terraform
# Copyright (c) 2025 Axians Infoma GmbH
# SPDX-License-Identifier: MPL-2.0

# Manages an explicitly scheduled upgrade for a Business Central environment.
# Use this resource when you need full control over the target version and schedule.
# Do NOT use application_version on bcadmincenter_environment for the same environment.

terraform {
  required_providers {
    bcadmincenter = {
      source = "axiansinfoma/bcadmincenter"
    }
  }
}

resource "bcadmincenter_environment" "prod" {
  name               = "my-production"
  application_family = "BusinessCentral"
  type               = "Production"
  country_code       = "US"
  ring_name          = "PROD"
  # application_version intentionally omitted — managed via update_schedule below
}

resource "bcadmincenter_environment_update_schedule" "prod_upgrade" {
  application_family   = bcadmincenter_environment.prod.application_family
  environment_name     = bcadmincenter_environment.prod.name
  target_version       = "26.2"
  scheduled_datetime   = "2026-04-01T02:00:00Z"
  ignore_update_window = false
}
```

### Update Schedule Without Explicit Datetime

```terraform
resource "bcadmincenter_environment" "prod" {
  name               = "my-production"
  application_family = "BusinessCentral"
  type               = "Production"
  country_code       = "US"
  ring_name          = "PROD"
  # application_version intentionally omitted — managed via update_schedule
}

resource "bcadmincenter_environment_update_schedule" "prod_upgrade" {
  application_family = bcadmincenter_environment.prod.application_family
  environment_name   = bcadmincenter_environment.prod.name
  target_version     = "26.2"
  # scheduled_datetime omitted: upgrade runs in next update window
}
```

### Bypassing the Update Window

```terraform
resource "bcadmincenter_environment_update_schedule" "emergency_upgrade" {
  application_family   = "BusinessCentral"
  environment_name     = "my-production"
  target_version       = "26.2"
  scheduled_datetime   = "2026-04-01T02:00:00Z"
  ignore_update_window = true  # may start at scheduled_datetime regardless of update window
}
```

## Schema

### Required

- `application_family` (String) The application family for the environment (e.g. `"BusinessCentral"`).
- `environment_name` (String) The name of the target environment.
- `target_version` (String) The target application version to upgrade to (e.g. `"26.1"`). Must match a value from the environment's available updates.

### Optional

- `aad_tenant_id` (String) The Azure AD tenant ID. If not specified, the provider's configured tenant ID is used.
- `ignore_update_window` (Boolean) When `true`, the upgrade may start at `scheduled_datetime` even if outside the environment's configured update window. Defaults to `false`.
- `scheduled_datetime` (String) The RFC3339 datetime at which the upgrade should run. If omitted, the upgrade runs in the next update window.

### Read-Only

- `id` (String) The ARM-like resource ID for the update schedule.
- `latest_selectable_datetime` (String) The latest datetime the update can be scheduled to.
- `rollout_status` (String) The rollout status (e.g. `"Active"`, `"UnderMaintenance"`, `"Postponed"`).
- `update_status` (String) The current update status (e.g. `"scheduled"`, `"running"`, `"failed"`).

## Import

Business Central environment update schedules can be imported using their ARM-like resource ID:

```shell
terraform import bcadmincenter_environment_update_schedule.example "/tenants/00000000-0000-0000-0000-000000000000/providers/Microsoft.Dynamics365.BusinessCentral/applications/BusinessCentral/environments/production/updateSchedule"
```

The import ID format is:
```
/tenants/{tenantId}/providers/Microsoft.Dynamics365.BusinessCentral/applications/{applicationFamily}/environments/{environmentName}/updateSchedule
```

Where:
- `{tenantId}` - The Azure AD tenant ID
- `{applicationFamily}` - The application family (e.g., "BusinessCentral")
- `{environmentName}` - The name of the environment

## Drift Detection

The resource applies the following drift detection rules on `Read`:

| Condition | Terraform behavior |
|---|---|
| `update_status` is `"scheduled"` or `"running"` | No drift |
| `update_status` is `"failed"` | Drift — retrigger Create on next apply |
| No selected update exists in the API | Resource removed from state |

## Update Behavior

- If `target_version` changes: the provider sends a new PATCH with `"selected": true` for the new version. The API automatically deselects the previous version.
- If only `scheduled_datetime` or `ignore_update_window` changes (same `target_version`): the provider sends a PATCH **without** the `"selected"` field to update schedule details only.
