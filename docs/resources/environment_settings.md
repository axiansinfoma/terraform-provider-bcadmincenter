# Resource (bcadmincenter_environment_settings)

Manages Business Central environment settings including update windows, telemetry, security groups, and access controls.

Manages Business Central environment settings including update windows, Application Insights telemetry, security group access controls, Microsoft 365 license access, app update cadence, and partner access settings.

~> **Important:** This resource manages settings for an existing Business Central environment. The environment must exist before settings can be configured. Use the `bc_admin_center_environment` resource to create environments.

~> **Warning:** Setting the `app_insights_key` attribute triggers an **automatic environment restart**. Plan this operation during non-business hours to avoid disruption.

## Important Notes

- **Update Window:** Must be at least 6 hours long. The window defines when environment updates can start and run.
- **Time Zones:** Use Windows time zone identifiers (e.g., "Pacific Standard Time", "Eastern Standard Time"). Use the `bc_admin_center_timezones` data source to get valid values.
- **Application Insights:** Setting or changing the key causes an automatic environment restart.
- **M365 License Access:** Requires environment version 21.1 or later.
- **Partner Access:** Only internal global administrators can modify partner access settings.
- **Settings Persistence:** Deleting this resource from Terraform state does not reset settings to defaults; they remain as configured.

## Example Usage

### Basic Update Window Configuration

```terraform
# Copyright (c) 2025 Axians Infoma GmbH
# SPDX-License-Identifier: MPL-2.0

# Basic environment settings with update window configuration

terraform {
  required_providers {
    bcadmincenter = {
      source = "axiansinfoma/bcadmincenter"
    }
  }
}

resource "bc_admin_center_environment_settings" "production" {
  application_family = "BusinessCentral"
  environment_name   = "production"

  # Configure update window (must be at least 6 hours)
  update_window_start_time = "22:00" # 10 PM
  update_window_end_time   = "06:00" # 6 AM
  update_window_timezone   = "Pacific Standard Time"

  # Enable Application Insights telemetry
  # Note: Setting this triggers an automatic environment restart
  app_insights_key = "InstrumentationKey=your-app-insights-key;IngestionEndpoint=https://westus2-1.in.applicationinsights.azure.com/"

  # Configure app update cadence
  app_update_cadence = "DuringMajorUpgrade"
}

# Reference an environment resource
resource "bc_admin_center_environment" "sandbox" {
  name               = "sandbox-1"
  application_family = "BusinessCentral"
  type               = "Sandbox"
  country_code       = "US"
  ring_name          = "Production"
  azure_region       = "eastus"
}

resource "bc_admin_center_environment_settings" "sandbox" {
  application_family = bc_admin_center_environment.sandbox.application_family
  environment_name   = bc_admin_center_environment.sandbox.name

  update_window_start_time = "20:00"
  update_window_end_time   = "04:00"
  update_window_timezone   = "Eastern Standard Time"

  # Restrict access to specific Azure AD security group
  security_group_id = "12345678-1234-1234-1234-123456789012"

  # Enable M365 license access (requires BC 21.1+)
  access_with_m365_licenses = true
}
```

## Schema

### Required

- `application_family` (String) Family of the environment's application (e.g., 'BusinessCentral')
- `environment_name` (String) Name of the environment

### Optional

- `access_with_m365_licenses` (Boolean) Whether users can access the environment with Microsoft 365 licenses (requires environment version 21.1+). Note: This setting may not be available on all environments.
- `allowed_partner_tenant_ids` (List of String) List of partner tenant IDs allowed to access the environment. Only used when partner_access_status is 'AllowSelectedPartnerTenants'
- `app_insights_key` (String, Sensitive) Application Insights connection string or instrumentation key for environment telemetry. Warning: Setting this triggers an automatic environment restart.
- `app_update_cadence` (String) How frequently AppSource apps should be updated. Valid values: 'Default', 'DuringMajorUpgrade', 'DuringMajorMinorUpgrade'
- `partner_access_status` (String) Partner access configuration. Valid values: 'Disabled', 'AllowAllPartnerTenants', 'AllowSelectedPartnerTenants'. Note: Only internal global administrators can modify this setting.
- `security_group_id` (String) Microsoft Entra (Azure AD) security group object ID to restrict environment access
- `update_window_end_time` (String) End time for the update window in HH:mm format (24-hour). Requires update_window_timezone to be set. Must be at least 6 hours after start time.
- `update_window_start_time` (String) Start time for the update window in HH:mm format (24-hour). Requires update_window_timezone to be set.
- `update_window_timezone` (String) Windows time zone identifier for the update window (e.g., 'Pacific Standard Time', 'Eastern Standard Time'). Required if update_window_start_time or update_window_end_time are set.

### Read-Only

- `id` (String) ARM-like resource ID (format: /tenants/{tenantId}/providers/Microsoft.Dynamics365.BusinessCentral/applications/{applicationFamily}/environments/{environmentName}/settings)

## Best Practices

### Update Window Sizing

- Ensure the update window is large enough for your environment to update completely
- Larger environments with many apps may need windows longer than the 6-hour minimum
- Consider database size and number of per-tenant extensions when sizing windows

### Time Zone Selection

- Use time zones that match your business hours to avoid updates during peak usage
- For multi-region environments, choose a time zone appropriate for the primary user base
- Use the `bc_admin_center_timezones` data source to find valid identifiers

### Application Insights

- Always use a connection string (preferred) rather than legacy instrumentation keys
- Test telemetry configuration in sandbox environments first
- Schedule the restart during maintenance windows

### Security Groups

- Use dedicated security groups for each environment tier (prod, sandbox, etc.)
- Document group membership and purpose
- Regularly audit group membership

### App Update Cadence

- **Default**: Apps update on Microsoft's schedule
- **DuringMajorUpgrade**: Apps update only during major environment updates
- **DuringMajorMinorUpgrade**: Apps update during both major and minor updates
- Choose based on your change management processes and testing requirements

## Common Issues

### Update Window Conflicts

**Error:** `ScheduledUpgradeConstraintViolation`

This occurs when:
- The update window conflicts with a scheduled update date
- The update is scheduled for today but the window is in the past

**Solution:** Adjust the update window or reschedule the update to a compatible date.

### Application Insights Restart

**Issue:** Setting `app_insights_key` causes unexpected downtime

**Solution:** Always plan for and communicate environment restarts when changing this setting. Consider:
- Scheduling during maintenance windows
- Notifying users in advance
- Testing in sandbox environments first

### Partner Access Permission Denied

**Error:** Permission denied when setting `partner_access_status`

**Cause:** Only internal global administrators can modify partner access settings.

**Solution:** Ensure the service principal or user has Global Administrator role in Azure AD for the tenant.

### M365 License Access Not Available

**Error:** Feature not available when setting `access_with_m365_licenses`

**Cause:** Environment version is older than 21.1

**Solution:** Update the environment to version 21.1 or later before enabling this feature.
