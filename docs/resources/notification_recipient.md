# Resource (bcadmincenter_notification_recipient)

Manages a notification recipient for the Business Central tenant. Notification recipients receive emails about environment lifecycle events such as update availability, successful updates, update failures, and extension validations. Up to 100 notification recipients can be configured per tenant.

Notification recipients receive emails about environment lifecycle events from the Microsoft account `no-reply-dynamics365@microsoft.com`. These events include:

- Update availability notifications
- Successful update completions
- Update failures and issues
- Extension validation results
- Environment lifecycle changes

-> **Note:** You can configure up to 100 notification recipients per tenant. For more recipients, use a distribution list email address.

~> **Important:** Ensure notification emails are not filtered to spam folders. Add `no-reply-dynamics365@microsoft.com` to your email allowlist.

## Best Practices

### Recipient Management

1. **Use Dedicated Email Addresses**: Prefer team or role-based email addresses over personal emails
2. **Distribution Lists**: For large organizations, use distribution lists to manage many recipients
3. **Regular Reviews**: Periodically review and update the recipient list to ensure accuracy
4. **Avoid Personal Accounts**: Use organizational email addresses that persist beyond individual employees

### Notification Handling

1. **Email Filtering**: Configure email systems to allowlist `no-reply-dynamics365@microsoft.com`
2. **Email Routing**: Set up email rules to route notifications to appropriate ticketing systems
3. **Monitoring**: Ensure at least one administrator's email is always configured
4. **Backup Recipients**: Configure multiple recipients to ensure notifications are always received

### Alternative Notification Channels

In addition to email notifications configured through this resource, consider these complementary channels:

- **Microsoft 365 Message Center**: Posts most environment lifecycle events
- **Service Health Dashboard**: Monitor outages and service issues
- **Application Insights Telemetry**: Most complete event logging with automation support
- **Azure Logic Apps**: Custom notification workflows and automation

### Multi-Tenant Scenarios

When managing notifications across multiple tenants, you can specify the `aad_tenant_id` attribute:

```terraform
# Use variables for consistent recipient configuration across tenants
variable "notification_recipients" {
  description = "Notification recipients configuration"
  type = list(object({
    email = string
    name  = string
  }))
  default = [
    {
      email = "admin@example.com"
      name  = "Primary Administrator"
    },
    {
      email = "ops@example.com"
      name  = "Operations Team"
    }
  ]
}

# Configure recipients for the primary tenant (uses provider's tenant_id by default)
resource "bcadmincenter_notification_recipient" "primary_recipients" {
  for_each = { for idx, recipient in var.notification_recipients : idx => recipient }
  
  email = each.value.email
  name  = each.value.name
}

# Configure recipients for a secondary tenant
resource "bcadmincenter_notification_recipient" "secondary_recipients" {
  for_each = { for idx, recipient in var.notification_recipients : idx => recipient }
  
  aad_tenant_id = var.secondary_tenant_id
  email         = each.value.email
  name          = each.value.name
}
```

-> **Note:** When `aad_tenant_id` is not specified, the provider's configured `tenant_id` is used by default. This allows you to manage notifications across multiple tenants with a single service principal that has the appropriate permissions.

## Common Issues

### Emails Not Received

If notification emails are not being received:

1. **Check Spam Filters**: Verify `no-reply-dynamics365@microsoft.com` is not blocked
2. **Verify Email Address**: Ensure the email address is correct and active
3. **Distribution List Limits**: Verify distribution lists are configured to accept external emails
4. **Email Quotas**: Check if recipient mailboxes are full or over quota
