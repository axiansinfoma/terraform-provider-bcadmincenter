# Data Source (bcadmincenter_quotas)

Retrieves environment quotas and capacity information for the tenant, including production and sandbox environment limits and storage capacity.

This data source retrieves environment quota information for the tenant, including limits and current usage for production and sandbox environments, as well as storage capacity. Use this data source for capacity planning and to check available resources before creating new environments.

## Example Usage

### Basic Usage

```terraform
# Copyright (c) 2025 Michael Villani
# SPDX-License-Identifier: MPL-2.0

# Query tenant environment quotas

terraform {
  required_providers {
    bcadmincenter = {
      source = "vllni/bcadmincenter"
    }
  }
}

data "bcadmincenter_quotas" "tenant" {}

# Output quota information
output "environment_capacity" {
  value = {
    production = {
      quota     = data.bcadmincenter_quotas.tenant.production_environments_quota
      allocated = data.bcadmincenter_quotas.tenant.production_environments_allocated
      available = data.bcadmincenter_quotas.tenant.production_environments_available
    }
    sandbox = {
      quota     = data.bcadmincenter_quotas.tenant.sandbox_environments_quota
      allocated = data.bcadmincenter_quotas.tenant.sandbox_environments_allocated
      available = data.bcadmincenter_quotas.tenant.sandbox_environments_available
    }
    storage = {
      quota_gb     = data.bcadmincenter_quotas.tenant.storage_quota_gb
      allocated_gb = data.bcadmincenter_quotas.tenant.storage_allocated_gb
      available_gb = data.bcadmincenter_quotas.tenant.storage_available_gb
    }
  }
}

# Check if we have capacity before creating an environment
resource "bcadmincenter_environment" "conditional" {
  count = data.bcadmincenter_quotas.tenant.production_environments_available > 0 ? 1 : 0

  name               = "new-production"
  application_family = "BusinessCentral"
  type               = "Production"
  country_code       = "US"
  ring_name          = "PROD"
}
```

### Capacity Monitoring

```terraform
data "bcadmincenter_quotas" "tenant" {}

# Monitor capacity and trigger alerts
output "capacity_alert" {
  value = {
    production_low = data.bcadmincenter_quotas.tenant.production_environments_available < 2
    sandbox_low    = data.bcadmincenter_quotas.tenant.sandbox_environments_available < 5
    storage_low    = data.bcadmincenter_quotas.tenant.storage_available_gb < 10
  }
}

# Calculate utilization percentages
output "utilization" {
  value = {
    production_pct = (
      data.bcadmincenter_quotas.tenant.production_environments_allocated /
      data.bcadmincenter_quotas.tenant.production_environments_quota * 100
    )
    sandbox_pct = (
      data.bcadmincenter_quotas.tenant.sandbox_environments_allocated /
      data.bcadmincenter_quotas.tenant.sandbox_environments_quota * 100
    )
    storage_pct = (
      data.bcadmincenter_quotas.tenant.storage_allocated_gb /
      data.bcadmincenter_quotas.tenant.storage_quota_gb * 100
    )
  }
}
```

### Conditional Resource Creation

```terraform
data "bcadmincenter_quotas" "tenant" {}

# Only create environment if quota is available
resource "bcadmincenter_environment" "optional_sandbox" {
  count = data.bcadmincenter_quotas.tenant.sandbox_environments_available > 0 ? 1 : 0

  name               = "test-environment"
  application_family = "BusinessCentral"
  type               = "Sandbox"
  country_code       = "US"
  ring_name          = "PROD"
}
```

## Understanding Quotas

### Environment Quotas

The number of production and sandbox environments you can create depends on your Business Central subscription:

- **Production Environments**: Typically limited based on your license (usually 1-3)
- **Sandbox Environments**: More generous allocation (typically 3-10+)

### Storage Quotas

Storage quotas are shared across all environments in the tenant:

- Measured in gigabytes (GB)
- Includes database storage for all environments
- Automatic backups consume additional quota

### Quota Exceptions

You can temporarily exceed production environment quotas when restoring environments (limited to one extra production environment for 30 days). This exception does not apply to sandbox environments.

## Important Notes

~> **Quota Limits**: Creating resources that exceed available quotas will fail. Always check `*_available` attributes before creating new environments.

~> **Real-time Data**: Quota data reflects current usage but may not account for in-progress operations. Consider adding buffer capacity in automation scripts.
