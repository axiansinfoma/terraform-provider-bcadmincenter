# Data Source (bcadmincenter_notification_settings)

Retrieves the complete notification settings for the Business Central tenant, including the Azure AD tenant ID and all configured notification recipients. This data source provides read-only access to the tenant-wide notification configuration.

Use this data source to retrieve the complete notification configuration for your Business Central tenant, including all configured notification recipients. This is useful for:

- Viewing current notification recipients
- Making conditional decisions based on existing configuration
- Validating notification setup
- Cross-referencing recipient information

-> **Note:** This data source provides read-only access to tenant-wide notification settings. To manage individual recipients, use the `bcadmincenter_notification_recipient` resource.

## Example Usage

### Basic Usage

```terraform
# Copyright (c) 2025 Axians Infoma GmbH
# SPDX-License-Identifier: MPL-2.0

# Get notification settings for the tenant

terraform {
  required_providers {
    bcadmincenter = {
      source = "axiansinfoma/bcadmincenter"
    }
  }
}

data "bcadmincenter_notification_settings" "current" {
}

# Output the tenant ID and recipients
output "tenant_id" {
  value = data.bcadmincenter_notification_settings.current.aad_tenant_id
}

output "notification_recipients" {
  value = data.bcadmincenter_notification_settings.current.recipients
}
```

### Conditional Resource Creation

```terraform
data "bcadmincenter_notification_settings" "current" {
}

# Only create a new recipient if fewer than 10 exist
resource "bcadmincenter_notification_recipient" "backup" {
  count = length(data.bcadmincenter_notification_settings.current.recipients) < 10 ? 1 : 0
  
  email = "backup-admin@example.com"
  name  = "Backup Administrator"
}
```

### Check for Existing Recipients

```terraform
data "bcadmincenter_notification_settings" "current" {
}

locals {
  admin_emails = [for r in data.bcadmincenter_notification_settings.current.recipients : r.email]
  has_admin    = contains(local.admin_emails, "admin@example.com")
}

output "admin_configured" {
  value       = local.has_admin
  description = "Whether admin@example.com is configured as a recipient"
}

output "recipient_count" {
  value       = length(data.bcadmincenter_notification_settings.current.recipients)
  description = "Total number of notification recipients"
}
```

### List All Recipients

```terraform
data "bcadmincenter_notification_settings" "current" {
}

output "all_recipients" {
  value = [
    for recipient in data.bcadmincenter_notification_settings.current.recipients : {
      name  = recipient.name
      email = recipient.email
    }
  ]
  description = "All configured notification recipients"
}
```

### Validate Recipient Limit

```terraform
data "bcadmincenter_notification_settings" "current" {
}

# Ensure we don't exceed the 100 recipient limit
check "recipient_limit" {
  assert {
    condition = length(data.bcadmincenter_notification_settings.current.recipients) < 100
    error_message = "Recipient limit approaching maximum of 100"
  }
}
```

### Multi-Tenant Query

```terraform
# Query notification settings for the provider's default tenant
data "bcadmincenter_notification_settings" "primary_tenant" {
}

# Query notification settings for a different tenant
data "bcadmincenter_notification_settings" "secondary_tenant" {
  aad_tenant_id = var.secondary_tenant_id
}

output "tenant_comparison" {
  value = {
    primary_count   = length(data.bcadmincenter_notification_settings.primary_tenant.recipients)
    secondary_count = length(data.bcadmincenter_notification_settings.secondary_tenant.recipients)
  }
  description = "Comparison of notification recipients across tenants"
}
```

## Notes

### Data Freshness

The data source queries the Business Central Admin Center API each time Terraform runs. The data is always current as of the time of the Terraform operation.

### Tenant Context

The `aad_tenant_id` returned matches the tenant ID used for authentication. This confirms which tenant's notification settings are being retrieved.

### Recipient Count Limit

Business Central supports up to 100 notification recipients per tenant. If you need to notify more people, consider using a distribution list email address as one of the recipients.

### No Input Required

This data source does not require any input parameters. It automatically retrieves the notification settings for the authenticated tenant.

```terraform
}

# Alert if no recipients are configured
check "recipients_configured" {
  assert {
    condition     = length(data.bcadmincenter_notification_settings.current.recipients) > 0

  }

```

### Multi-Tenant Management

```terraform
# When managing multiple tenants, verify configuration across all
data "bcadmincenter_notification_settings" "current" {
}

locals {
  required_recipients = [
    "admin@example.com",
    "ops@example.com"
  ]
  
  configured_emails = [
    for r in data.bcadmincenter_notification_settings.current.recipients : r.email
  ]
  
  missing_recipients = [
    for email in local.required_recipients : email
    if !contains(local.configured_emails, email)
  ]
}

output "missing_recipients" {
  value       = local.missing_recipients
  description = "Required recipients not yet configured"
}
```
