# Data Source (bcadmincenter_environments)

Retrieves a list of all Business Central environments for a given application family.

This data source retrieves a list of all Business Central environments for a given application family. Use this to discover all environments in your tenant, filter them by type, or build dynamic configurations based on existing environments.

## Example Usage

### Basic Example

```terraform
# Copyright (c) 2025 Michael Villani
# SPDX-License-Identifier: MPL-2.0

# List all Business Central environments for a given application family

terraform {
  required_providers {
    bcadmincenter = {
      source = "vllni/bcadmincenter"
    }
  }
}

provider "bcadmincenter" {
  # Authentication configured via environment variables or provider block
}

data "bcadmincenter_environments" "all" {
  application_family = "BusinessCentral"
}

# Output all environment names
output "environment_names" {
  value = [for env in data.bcadmincenter_environments.all.environments : env.name]
}

# Filter for production environments
output "production_environments" {
  value = [for env in data.bcadmincenter_environments.all.environments : env.name if env.type == "Production"]
}

# Filter for sandbox environments
output "sandbox_environments" {
  value = [for env in data.bcadmincenter_environments.all.environments : env.name if env.type == "Sandbox"]
}

# Output environment URLs
output "environment_urls" {
  value = {
    for env in data.bcadmincenter_environments.all.environments :
    env.name => env.web_client_login_url
  }
}
```

### Finding Specific Environments

```terraform
data "bcadmincenter_environments" "all" {
  application_family = "BusinessCentral"
}

# Find a specific environment by name
locals {
  prod_env = [
    for env in data.bcadmincenter_environments.all.environments :
    env if env.name == "production"
  ][0]
}

output "production_url" {
  value = local.prod_env.web_client_login_url
}
```

### Creating Resources Dynamically

```terraform
data "bcadmincenter_environments" "all" {
  application_family = "BusinessCentral"
}

# Create a monitoring alert for each production environment
resource "monitoring_alert" "env_alerts" {
  for_each = {
    for env in data.bcadmincenter_environments.all.environments :
    env.name => env if env.type == "Production"
  }

  name   = "bc-${each.key}-alert"
  target = each.value.web_client_login_url
  status = each.value.status
}
```

## Best Practices

### Caching and Performance

The environments list is fetched from the API each time Terraform refreshes state. For large deployments, consider:

- Using `terraform refresh` selectively
- Implementing external caching mechanisms for CI/CD pipelines
- Filtering environments early in your configuration

### Error Handling

```terraform
data "bcadmincenter_environments" "all" {
  application_family = "BusinessCentral"
}

# Safely handle cases where no environments exist
locals {
  has_production = length([
    for env in data.bcadmincenter_environments.all.environments :
    env if env.type == "Production"
  ]) > 0
}

output "production_exists" {
  value = local.has_production
}
```

## Common Issues

### Empty Environment List

If no environments are returned, verify:
- Your Azure AD application has proper permissions
- The application is added to the **AdminAgents** group
- The `application_family` value is correct (typically "BusinessCentral")

### Permission Denied

Ensure your Azure AD application has the `AdminCenter.ReadWrite.All` permission on the "Dynamics 365 Business Central administration center" API.

### Rate Limiting

The Business Central Admin Center API has rate limits. If you're making frequent requests, implement appropriate retry logic and delays in your automation.
