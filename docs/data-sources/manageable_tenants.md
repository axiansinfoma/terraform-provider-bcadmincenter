# Data Source (bcadmincenter_manageable_tenants)

Retrieves a list of Microsoft Entra tenants that the authenticating app can manage. Note: This data source can only be used when authenticated as an app (service principal), not with user authentication.

Use this data source to retrieve all Microsoft Entra tenants where the authenticating app is authorized to manage Business Central environments. This is particularly useful for multi-tenant applications that need to discover which customer tenants they have access to.

~> **Important:** This data source can ONLY be used when the provider is authenticated as an app (service principal) using client credentials flow. It will fail if used with user authentication (delegated permissions).

## Multi-Tenant App Workflow

For Microsoft Entra apps serving multiple customers:

1. **App is registered** in the app vendor's tenant
2. **App is configured as multi-tenant** in Azure AD
3. **Each customer authorizes the app** in their Business Central Admin Center
4. **App uses this data source** to discover all customer tenants it can manage

This enables apps to dynamically discover their authorized tenants without hardcoding tenant IDs.

## Example Usage

### List All Manageable Tenants

```terraform
# Copyright (c) 2025 Axians Infoma GmbH
# SPDX-License-Identifier: MPL-2.0

# Example: Get Manageable Tenants for App
# Note: This data source can only be used when authenticated as an app (service principal)

terraform {
  required_providers {
    bcadmincenter = {
      source = "axiansinfoma/bcadmincenter"
    }
  }
}

provider "bcadmincenter" {
  # Authentication must use app credentials (client credentials flow)
  # AZURE_CLIENT_ID
  # AZURE_CLIENT_SECRET
  # AZURE_TENANT_ID
}

# Get a list of all tenants where this app is authorized
data "bcadmincenter_manageable_tenants" "all" {}

# Output the list of manageable tenant IDs
output "manageable_tenant_ids" {
  description = "List of tenant IDs where this app can manage Business Central"
  value       = [for tenant in data.bcadmincenter_manageable_tenants.all.tenants : tenant.entra_tenant_id]
}

# Output the number of manageable tenants
output "tenant_count" {
  description = "Number of tenants this app can manage"
  value       = length(data.bcadmincenter_manageable_tenants.all.tenants)
}
```

## Authentication Requirements

~> **Critical:** This data source requires app authentication (client credentials flow). It CANNOT be used with user authentication.

The provider must be configured with:
- `client_id` - The application (client) ID
- `client_secret` - The application secret (or use certificate-based auth)
- `tenant_id` - Any tenant where the app is registered (typically the app vendor's tenant)

For multi-tenant apps, the `tenant_id` should be the home tenant where the app is registered.

## API Behavior

This endpoint returns tenants where:
1. The app is authorized in the Business Central Admin Center
2. The app MAY OR MAY NOT have admin consent for `AdminCenter.ReadWrite.All`

The presence of a tenant in this list indicates authorization in the Admin Center, but you should still handle API permission errors gracefully in case admin consent has not been granted.

## Admin Consent Requirement

The data source will return tenants where the app is authorized in the Admin Center, but those tenants may not have granted admin consent for the `AdminCenter.ReadWrite.All` permission.

When calling other Admin Center API endpoints for these tenants, you may encounter permission errors if admin consent has not been granted. Always implement proper error handling.

## Performance Considerations

This data source makes a single API call that returns all manageable tenants. For apps with many authorized tenants:

- The response includes only tenant IDs (minimal data)
- Results can be cached using Terraform's standard mechanisms
- Consider using `lifecycle` blocks to prevent unnecessary re-reads

## Troubleshooting

### "This endpoint can only be used when authenticated as an app" Error

**Cause:** The provider is configured with user credentials or delegated permissions.

**Solution:** Configure the provider with app credentials:
```terraform
provider "bcadmincenter" {
  client_id     = "your-app-id"
  client_secret = "your-app-secret"
  tenant_id     = "your-home-tenant-id"
}
```

### Empty Tenant List

**Possible Causes:**
1. The app has not been authorized in any tenant's Admin Center
2. The app is using the wrong credentials
3. The app is registered as single-tenant instead of multi-tenant

**Solutions:**
1. Ensure the app is authorized in at least one tenant
2. Verify the `client_id` and `client_secret` are correct
3. Check the app registration in Azure AD and set it to multi-tenant if needed

### Permission Denied When Accessing Tenant

**Cause:** Tenant appears in the list but admin consent has not been granted.

**Solution:** Have the tenant administrator grant admin consent for the `AdminCenter.ReadWrite.All` permission.

## Security Best Practices

1. **Limit app permissions** - Only grant necessary API permissions
2. **Rotate secrets regularly** - Update client secrets on a schedule
3. **Use certificates** - Prefer certificate-based authentication over client secrets
4. **Monitor tenant access** - Regularly audit which tenants the app can access
5. **Implement logging** - Log all tenant access and API operations
6. **Validate tenants** - Verify tenant IDs before performing operations
