# Data Source (bcadmincenter_environment)

Retrieves information about a specific Business Central environment.

This data source retrieves information about a specific Business Central environment within an application family. Use this to query environment details such as status, version, URLs, and configuration.

## Example Usage

### Basic Example

```terraform
# Copyright (c) 2025 Michael Villani
# SPDX-License-Identifier: MPL-2.0

# Get information about a specific Business Central environment

terraform {
  required_providers {
    bcadmincenter = {
      source = "vllni/bcadmincenter"
    }
  }
}

provider "bcadmincenter" {
  # Authentication configured via environment variables or provider block
}

data "bcadmincenter_environment" "example" {
  application_family = "BusinessCentral"
  name               = "my-production-env"
}

# Use the retrieved environment information
output "environment_status" {
  value = data.bcadmincenter_environment.example.status
}

output "web_client_url" {
  value = data.bcadmincenter_environment.example.web_client_login_url
}

output "application_version" {
  value = data.bcadmincenter_environment.example.application_version
}
```

### Using with Resource

```terraform
# Get information about an existing environment
data "bcadmincenter_environment" "existing" {
  application_family = "BusinessCentral"
  name               = "my-env"
}

# Use the data in conditional logic
resource "bcadmincenter_environment" "new_sandbox" {
  count = data.bcadmincenter_environment.existing.type == "Production" ? 1 : 0

  application_family = "BusinessCentral"
  name               = "${data.bcadmincenter_environment.existing.name}-sandbox"
  type               = "Sandbox"
  country_code       = data.bcadmincenter_environment.existing.country_code
  ring_name          = data.bcadmincenter_environment.existing.ring_name
  azure_region       = data.bcadmincenter_environment.existing.azure_region
}
```

## Use Cases

### Environment Status Monitoring

Query environment status to integrate with monitoring systems:

```terraform
data "bcadmincenter_environment" "prod" {
  application_family = "BusinessCentral"
  name               = "production"
}

output "prod_status" {
  value = {
    status             = data.bcadmincenter_environment.prod.status
    version            = data.bcadmincenter_environment.prod.application_version
    web_client_url     = data.bcadmincenter_environment.prod.web_client_login_url
  }

```

Use environment details to create similar environments:

```terraform

  application_family = "BusinessCentral"

```

## Common Issues

### Environment Not Found

If the environment doesn't exist, the data source will return an error. Ensure the environment name is correct and that your authentication has access to view it.

### Permission Denied

Ensure your Azure AD application has the `AdminCenter.ReadWrite.All` permission and is added to the **AdminAgents** group for the Business Central tenant.
