### Problem Statement
Current documentation examples are not yet structured for smooth onboarding and advanced usage scenarios.

Key gaps:
- Authentication examples are grouped instead of split by authentication method.
- There is no tutorial-oriented flow similar to what users expect from mature providers like Azure.
- No end-to-end environment tutorial shows how to derive values from data sources (e.g., application family, ring, etc.).
- Multi-tenant scenarios are not covered in a practical tutorial (including iteration and import workflow).

### Goal
Improve documentation quality and usability with structured tutorials and focused examples that users can directly copy and adapt.

### Scope (This Issue)
1. Split authentication examples: Separate examples per auth strategy and keep each concise.
2. Explore and adopt tutorial structure inspired by Azure provider: review Azure tutorial patterns, introduce separation between reference docs and guided tutorials.
3. Create a full environment tutorial: add a tutorial for provisioning/managing an environment reading properties from data sources where possible.
4. Create a multi-tenant tutorial: use available tenants data, iterate to create a sandbox in each, include import statements.

### Non-Goals
- Unrelated provider runtime refactors.
- Changes to unrelated resources not needed for docs/tutorial deliverables.

### Acceptance Criteria
- [ ] Authentication examples are split and clearly organized by method.
- [ ] A tutorial-oriented docs flow is added, inspired by Azure provider patterns.
- [ ] A full environment tutorial exists and uses data sources for dynamic inputs where possible.
- [ ] A multi-tenant tutorial exists using tenant iteration and import statements.
- [ ] All new docs are copy/paste-ready and aligned with Terraform best practices.
- [ ] Generated/reference docs remain consistent with repository standards.

### Additional Context
Use Azure provider tutorials for inspiration; adapt content and structure to this provider’s capabilities and conventions.
